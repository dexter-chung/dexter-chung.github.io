# Configuration Management

The first thing that I learned from this course is Configuration Management which is basically the process of maintaining the functionality of a project through tracking changes, looking at different versions, and making sure the correct files are being worked on. This usage of Configuration Management can be seen through the use of Github with their branches which helps in making different versions of project with out changing the main files which helps in preventing files from being overridden.

# Agile Project Management

The second thing that I learned from this course is Agile Project Management in which is the process of having a flexable approach to projects and being to adapt to changes when needed. When planning for a big project sometimes not ever aspect of that plan will come to fruition for example having to face time constraints. In those times, it is important to be able to prioritize what is actaully needed and what is not to be able to make the finished product. In Agile Project Management a project can be broken down into smaller simpler tasks which can help in organization. 

# Design Patterns

Lastly the other thing I learned from this course is the approach of Design Patterns. In which is the approach of making solutions to commonly occuring problems. Some commonly occuring problems could be installing the correct dependencies for making a new project. It helps get rid of the tedious process of having to come up with a solution yourself when one already exists.
